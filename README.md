# Rolldown Benchmark

## Notes

### Source apps

- Apps containing a mix of React JSX components and plain JS from `node_modules`, with total modules ranging from 2.4k to 19k:
  - `apps/1000`: 2413 modules(1000 JSX components + 1413 JS modules in node_modules)
  - `apps/3000`: 5714 modules(3000 JSX components + 2714 JS modules in node_modules)
  - `apps/5000`: 9014 modules(5000 JSX components + 4014 JS modules in node_modules)
  - `apps/10000`: 19014 modules(10000 JSX components + 9014 JS modules in node_modules)
- The original esbuild `three10x` benchmark
- `rome` based on https://github.com/rome/tools/tree/archived-js, total 1195 typescript file

### Configurations

Right now, Rolldown's built-in minification (oxc-minify) is not feature-complete yet, so we separate the benchmarks into the following categories:

- Pure bundling, without minification and source maps
- Bundling + minification + source maps (using SWC minifier)
  - Slow minifier, but better production bundle size
- Bundling + minification + source maps (using esbuild minifier)
  - Fast minifier, less optimized bundle size

The configurations with minifiers are focused on comparing the minifier overhead and final production bundle size using external minifiers. Our eventual goal is for Rolldown's built-in minifier to have the quality of SWC but the speed of esbuild.

### Environments

This is the default environment benchmarks are run in unless otherwise noted:

- OS: Linux Ubuntu 22.04 LTS
- CPU: (32) x64 AMD Ryzen 9 5950X 16-Core Processor
- Memory: 64 GB

Some benchmarks have additional runs on other environments for reference and comparing perf on systems with different number of CPU cores.
